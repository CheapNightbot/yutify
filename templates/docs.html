Endpoint

URL: /api/{artist}:{song}
Method: GET
Description: Retrieves music information for a specific artist and song. Replace {artist} and {song} with the actual artist and song name, respectively.

Query Parameters

  • platform (optional): Filters the music information by a specific platform. Possible values: deezer, itunes, spotify, ytmusic. If not provided, all available platform information is returned.
    • Example: /api/{artist}:{song}?spotify

Response

Status Code: 200 OK
Body: JSON object containing the following fields:

Schema

Field Description
album_art (string) URL to the album art.
album_title (string) Title of the album.
album_type (string) Type of the album (e.g., "single", "album").
artists (string) Name of the artist(s).
deezer (string | null) URL to the track on Deezer, or null if not available.
genre (string | null) Genre of the music, or null.
itunes (string | null) URL to the track on iTunes, or null if not available.
lyrics (string | null) Lyrics of the song, or null if not available.
release_date (string) Release date of the song in YYYY-MM-DD format.
spotify (string | null) URL to the track on Spotify, or null if not available.
title (string) Title of the song.
ytmusic (object | null)
Object containing the YouTube Music information, or null if not available. If available, contains:
id (string) ID of the song on YouTube Music.
url (string) URL to the track on YouTube Music.

Status Code: 404 NOT FOUND
Body: JSON object containing the following fields:

Schema

Field Description
error (string) Error message describing the issue.
